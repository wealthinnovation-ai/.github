# Contributing to Stream Kinetics

## Development Workflow (Strict TDD)

1. **Branch from `staging`** using the Linear branch name: `Cmd+Shift+.` in Linear
2. **Implement** your changes
3. **Quality Gate** — all must pass with zero warnings/errors:
   ```bash
   bun run lint
   bun run check
   bun run format
   bun run test:backend
   bun run test:e2e
   bun run test:smoke
   bun run test:unit
   ```
4. **Commit & Push** — use conventional commits:
   ```
   feat(chat): add WebSocket reconnect logic
   fix(auth): resolve session expiry redirect
   docs(api): update endpoint reference
   ```
5. **Open PR** — include Linear issue ID in title (e.g., `HAR-7: Scaffold PWA shell`)
6. **Review** — CODEOWNERS will auto-assign reviewers
7. **Merge** — squash merge to `staging`, Linear auto-updates issue status

## Package Manager

**Bun only.** Do not use `npm`, `npx`, `yarn`, or `pnpm`.

```bash
bun install          # Install dependencies
bun run dev          # Start dev server
bun run build        # Production build
bunx <package>       # Run package binaries
```

## Branch Naming

Branches are auto-generated by Linear:
```
{user}/{ISSUE-ID}-{description}
```
Example: `joshua/har-7-scaffold-pwa-shell`

## Code Style

- **TypeScript** — strict mode, no `any`
- **Svelte 5** — Runes (`$state`, `$derived`, `$effect`), no legacy stores
- **CSS** — Tailwind + Skeleton.dev tokens
- **Testing** — Vitest, 80%+ coverage target
